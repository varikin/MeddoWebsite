What

A menu bar app for Mac that lists entries in the /etc/hosts file and lets you enable and disable them.

Where

Download Meddo or check it out of Github. If you find a problem, please let me know.

Why

I have several entries for different staging servers that I need to frequently enable and disable. I wanted this easier.

How

Reading /etc/hosts

The first part of the app is parsing /etc/hosts. The general idea is to look at a line and determine if it is a comment, host entry, commented out host entry, or blank line. Host entries are grouped into one item by the idea of zero or more comments followed by one or more host entries. Host groups are separated by comments (not commented out host entries) and blank lines. A name for the group is determined by either the first non blank comment (octothropes are ignored) or the first host name.

This works well for my setup, but I am open to enhancements based on anyone elses setup. Send me a hosts file that is problematic and I will work on it.

Meddo watches for changes to /etc/hosts and updated the menu to reflect the changes.

Writing /etc/hosts

/etc/hosts is owned by root. Meddo runs as the user. To handle this issue, Apple provides a way to install helper tools that run as root. Meddo includes a helper tool: 

  /Library/PrivilegedHelperTools/com.fictitiousnonsense.MeddoHelper
  /Library/LaunchDaemons/com.fictitiousnonsense.MeddoHelper.plist

When writing to /etc/hosts, Meddo checks the version of the helper and installs a newer version if needed. Installing the helper is the only time the user's password is needed. All the helper does is write the file and send/recieve messages about writing the file. Everything else is done executed as the user.
